Rapid Batch-process-supported Powerful tool for GWAS & GS

基于混合线性模型（MLM）的大规模基因组数据高效 GWAS 和 GS 分析工具，支持批量性状分析。

1. 软件介绍

RBP（Rapid Batch-process-supported Powerful tool）是一款面向大规模基因组数据的高性能计算工具， 基于混合线性模型（Mixed Linear Model, MLM）框架，用于：

  • GWAS（全基因组关联分析）—— 扫描基因组中与性状显著关联的 SNP 位点
  • GS（基因组选择）—— 估计育种值（GEBV / Random Effects）用于个体选育

RBP 的核心特点：

  • 高性能：基于 C++ + Armadillo + OpenMP 实现，支持多线程并行
  • 双精度支持：同时支持 float（单精度）和 double（双精度）计算，单精度可大幅节省内存
  • Out-of-Core 分块模式：对超大规模数据（个体数 > 50,000）支持分块映射，突破内存限制
  • 批量性状分析：支持一次运行同时对多个性状进行方差组分估计和 GWAS 扫描
  • 多随机效应：支持构建多个 GRM 作为随机效应，以及额外分类随机效应
  • 多种 GWAS 检验方法：Standard MLM、Quasi-Standard、Gamma 近似、Exact MLM

RBP 提供了四个子命令（模块）：

  1. --make-grm — 构建基因组关系矩阵（GRM）
  2. --kpca — 基于 GRM 的主成分分析（kPCA）
  3. --single-trait — 单性状 GWAS / AI-REML 分析
  4. --batch-trait — 批量多性状 GWAS / DFREML 分析

2. 输入文件格式

2.1 表型文件（Phenotype File）

表型文件为纯文本文件，使用空格、制表符（Tab）或逗号分隔各列。

必须满足：文件必须包含表头（header），且第一列为个体 ID。

文件示例（phe.txt）：

phe.txt
ID          Trait1   Trait2   Sex    Batch   Age    Sire
IND001      12.5     8.3      M      B1      2.1    BULL_A
IND002      14.1     9.1      F      B2      1.8    BULL_B
IND003      11.8     7.9      M      B1      2.4    BULL_A
IND004      13.2     8.7      F      B1      1.5    BULL_C
IND005      10.9     7.2      M      B2      2.0    BULL_B
...

与此表型文件相关的常用参数：

参数说明示例
--phe FILE 指定表型文件路径 --phe phe.txt
--phe-col STR 指定作为响应变量（性状）的列，1-based 索引，逗号分隔。单性状模式仅取第一列；批量模式可取多列 --phe-col 2 或 --phe-col 2,3
--fix-col STR 指定分类固定效应列（如性别、批次），1-based，逗号分隔。程序自动进行哑变量编码（最后一个水平作为参照） --fix-col 4,5
--cov-col STR 指定连续协变量列（如年龄、体重），1-based，逗号分隔 --cov-col 6
--rand-col STR 指定额外随机效应的分类变量所在列，1-based，逗号分隔（仅单性状模式支持） --rand-col 7
--rand-cov LIST 指定额外随机效应对应的协方差矩阵文件前缀（空格分隔）。若为 1 则表示使用单位矩阵（即独立同分布随机效应）。也可传入 .txt 列表文件 --rand-cov pedigree_grm 或 --rand-cov 1
列索引说明：所有列索引（--phe-col、--fix-col、--cov-col、--rand-col）均为 1-based，即第一列（ID 列）为 1。

2.2 基因型文件（Genotype File）

基因型文件采用 PLINK 二进制格式（.bed / .bim / .fam），通过 --geno 参数指定前缀（不含扩展名，或含 .bed 后缀均可）。

用于指定读取个体和 SNP 子集的参数：

参数说明示例
--geno FILE 基因型数据 PLINK 前缀（或 .bed 完整路径） --geno ./data/geno
--geno-ind STR 指定要读取的个体索引（1-based），支持逗号分隔的列表 或 一个每行一个数字的文件路径 --geno-ind 1,3,5,100 或 --geno-ind keep_ids.txt
--geno-snp STR 指定要读取的 SNP 索引（1-based），支持逗号分隔的列表 或 一个每行一个数字的文件路径 --geno-snp 1,2,3,500 或 --geno-snp snp_list.txt
智能解析：--geno-ind 和 --geno-snp 会先尝试将输入作为文件路径打开；若文件不存在，则按逗号分隔的列表解析。索引均为 1-based。

3. 模块说明

RBP 四个模块通过第一个位置参数选择。以下通用参数适用于所有模块：

参数说明默认值
--out PREFIX输出文件前缀rbp
--ncpus NCPU 线程数1
--float使用单精度（float）计算以节省内存。GRM 构建时若启用，后续分析也必须启用关闭（使用 double）
--chunk启用 Out-of-Core 分块模式（推荐个体数 > 50,000）关闭
--quiet静默模式，不向终端打印日志（日志仍写入 .log 文件）关闭
重要：--float 参数的一致性
如果 GRM 构建时使用了 --float，那么后续的 kPCA、单性状分析等模块也必须使用 --float。因为 float 和 double 模式的 GRM 二进制文件数据类型不一致，混用会导致读取错误。

3.1 GRM 构建（--make-grm）

功能说明

从 PLINK 二进制格式的基因型数据构建基因组关系矩阵（Genomic Relationship Matrix, GRM）。 支持三种计算方法，并可选择是否计算 GRM 的广义逆矩阵。

输出文件

扩展名内容
.GA.bin加性 GRM 二进制矩阵（Yang 或 VanRaden 方法）
.GD.bin显性 GRM 二进制矩阵（Dominance 方法）
.id与 GRM 行列对应的个体 ID 列表
.inv.txtGRM 的广义逆矩阵，三列格式（行、列、值）—— 仅当指定 --inv 时生成

专属参数

参数说明默认值
--method NGRM 构建方法：1=Yang 加性（默认），2=VanRaden 加性，3=Dominance 显性1
--weight FILESNP 权重文件（单列文本，每行对应一个 SNP 的权重）无（所有 SNP 等权）
--snp-chunk N每次处理的 SNP 块大小，影响内存使用和计算效率10000
--ind-chunk N分块模式下每次处理的个体块大小（仅 --chunk 模式下生效）512
--inv计算 GRM 的广义逆矩阵并保存为三列格式 .inv.txt关闭
--geno-ind STR选择参与 GRM 构建的个体子集全部个体
--geno-snp STR选择参与 GRM 构建的 SNP 子集全部 SNP

完整示例

# 使用默认 Yang 方法构建加性 GRM（双精度）
RBP --make-grm --geno ./data/geno --out my_grm --ncpus 8

# 使用 VanRaden 方法 + SNP 权重（单精度，适用于超大数据集）
RBP --make-grm --geno ./data/geno --out my_grm_vanraden --method 2 --weight snp_weights.txt --float --ncpus 16

# 构建显性效应 GRM 并计算逆矩阵
RBP --make-grm --geno ./data/geno --out dom_grm --method 3 --inv --ncpus 8

# 使用指定个体子集构建 GRM，启用分块模式（个体 > 50,000）
RBP --make-grm --geno ./data/geno --out large_grm --geno-ind keep_list.txt --chunk --ind-chunk 1024 --ncpus 24

3.2 kPCA 分析（--kpca）

功能说明

基于已构建的 GRM，使用 Lanczos 算法计算前 k 个最大特征值及对应的特征向量（即主成分，PCs）。 kPCA 常用于群体分层检测以及将主成分作为协变量引入 GWAS 模型以控制群体结构。

输出文件

扩展名内容
.val特征值列表（每行一个）
.vec主成分（PCs）矩阵：第一列为个体 ID，后续列为各 PC 得分
注意：若提供了多个 GRM（--grm file1 file2），kPCA 模块仅对第一个 GRM 进行分析，并会输出警告提示。

专属参数

参数说明默认值
--grm PREFIX输入的 GRM 文件前缀（不含 .bin 扩展名）—（必填）
--npc N提取的主成分个数10
--tol VALLanczos 算法收敛容差1e-6

完整示例

# 从 GRM 中提取前 10 个主成分
RBP --kpca --grm my_grm --npc 10 --ncpus 8

# 指定输出前缀、提取前 20 个 PC，并使用单精度
RBP --kpca --grm my_grm_float --npc 20 --out pca_result --float --ncpus 8

3.3 单性状分析（--single-trait）

功能说明

对单个性状拟合混合线性模型，使用AI-REML 算法（Average Information Restricted Maximum Likelihood）估计方差组分， 并可选择进行 GWAS 扫描或计算随机效应预测值（育种值）及其 PEV（预测误差方差）。

单性状分析支持多种复杂模型结构：

  • 一般 GS / GWAS（单 GRM + 残差）
  • 重复力模型（repeated measures，使用 --repeat）
  • 多 GRM 随机效应（如加性 + 显性 + 上位）
  • 含额外分类随机效应（如批次、母体效应）
  • 以父母祖先遗传随机效应为背景（使用 --rand-col + --rand-cov）

输出文件

文件内容
.var方差组分估计（Variance、SE、Proportion）
.beta固定效应估计值及标准误
.rand所有随机效应的预测值（育种值等），可选 PEV 和可靠性 r²
.mlmGWAS 扫描结果（SNP、Chr、Pos、Effect、SE、Pvalue）—— 仅当指定 --gwas 时生成
.log运行日志（包含 Lambda GC 和 Mean Chi² 统计量）

专属参数

参数说明默认值
--grm LIST一个或多个 GRM 文件前缀（空格分隔）。也可传入 .txt 列表文件。所有 GRM 的数据类型（float/double）必须一致—
--rand-col STR额外随机效应的分类变量列—
--rand-cov LIST额外随机效应的协方差矩阵文件前缀（空格分隔）。若为 1 表示独立同分布—
--test METHODGWAS 检验方法：standard（标准 MLM）、quasi-standard（拟标准，不适用于 batch）、gamma（Gamma 近似）、exact（精确检验，不适用于 single）standard
--gwas执行 GWAS 扫描关闭
--solver METHODAI-REML 求解器：CG（共轭梯度，默认）或 CholeskyCG
--mom启用 MoM-RHE 初始化方差组分（加速收敛）关闭
--pev计算随机效应预测的 PEV 和可靠性（r²）关闭
--repeat指定重复力模型（repeated measures）关闭
--seed N随机数种子6666
--num-ranvec Nquasi-standard/gamma 方法使用的随机向量数100
--re-gamma-size N估计 gamma 因子时随机采样的 SNP 数量500
--re-est-gamma强制重新估计 gamma 因子关闭
--maxit NAI-REML 最大迭代次数30
--tol VALAI-REML 收敛容差1e-5
--cg-maxit NCG 求解器最大迭代次数500
--cg-tol VALCG 求解器收敛容差1e-6

关于 --grm 参数的详细用法

多 GRM 数据类型必须一致：当指定多个 GRM 时，所有 GRM 必须是同一精度（全部 float 或全部 double）构建的，否则会导致数据读取错误。

指定多个 GRM 的三种方式：

# 方式 1：直接列出多个 GRM 前缀（空格分隔）
RBP --single-trait --grm grm_add grm_dom grm_epi --phe phe.txt --phe-col 2 --geno geno

# 方式 2：通过 .txt 列表文件指定（每行一个 GRM 前缀）
# 文件 grm_list.txt 内容：
#   grm_add
#   grm_dom
#   grm_epi
RBP --single-trait --grm grm_list.txt --phe phe.txt --phe-col 2 --geno geno

# 方式 3：混合使用（直接前缀 + 列表文件）
RBP --single-trait --grm grm_add grm_list.txt --phe phe.txt --phe-col 2 --geno geno

GRM 的数据类型要求：

  • 所有 GRM 必须使用相同精度构建（全部 float 或全部 double）
  • 如果 GRM 构建时使用了 --float，单性状分析时也必须使用 --float
  • 每个 GRM 目录下需包含 .bin 和 .id 文件；若为压缩矩阵还需 .col 文件

完整示例

场景 1：一般 GS / GWAS（单 GRM）

# 仅估计方差组分和育种值（GS）
RBP --single-trait \
  --grm my_grm \
  --phe phe.txt \
  --phe-col 2 \
  --fix-col 4 \
  --cov-col 6 \
  --geno ./data/geno \
  --out gs_result \
  --ncpus 8

# GWAS 扫描（Standard MLM）
RBP --single-trait \
  --grm my_grm \
  --phe phe.txt \
  --phe-col 2 \
  --fix-col 4 \
  --cov-col 6 \
  --geno ./data/geno \
  --gwas \
  --out gwas_result \
  --ncpus 8

# GS + GWAS 同时进行
RBP --single-trait \
  --grm my_grm \
  --phe phe.txt \
  --phe-col 2 \
  --fix-col 4 \
  --cov-col 6 \
  --geno ./data/geno \
  --gwas --pev \
  --out gs_gwas_result \
  --ncpus 8

场景 2：重复力性状的 GS / GWAS

# 对同一性状有重复测量记录时，使用 --repeat 指定重复力模型
RBP --single-trait \
  --grm my_grm \
  --phe phe_repeat.txt \
  --phe-col 2 \
  --fix-col 4,5 \
  --geno ./data/geno \
  --gwas --repeat \
  --out repeat_result \
  --ncpus 8

场景 3：包含多 GRM 随机效应的 GS / GWAS

# 将加性和显性 GRM 同时纳入模型
RBP --single-trait \
  --grm grm_add grm_dom \
  --phe phe.txt \
  --phe-col 2 \
  --fix-col 4 \
  --cov-col 6 \
  --geno ./data/geno \
  --gwas --pev \
  --out multi_grm_result \
  --ncpus 8

场景 4：以父母祖先遗传随机效应为背景的 GS / GWAS

# --rand-col 指定表型文件中个体与父母对应关系的列索引
# --rand-cov 指定父母的亲缘关系矩阵
RBP --single-trait \
  --grm my_grm \
  --phe phe_with_sire.txt \
  --phe-col 3 \
  --fix-col 5 \
  --rand-col 7 \
  --rand-cov sire_grm \
  --geno ./data/geno \
  --gwas --pev \
  --out sire_model_result \
  --ncpus 8
父母模型示例说明：表型文件中第 7 列（--rand-col 7）记录了每个个体对应的父亲 ID； sire_grm 是预先构建的父亲间的亲缘关系矩阵（GRM）。模型将父亲遗传效应作为额外的随机效应纳入， 从而在估计个体加性效应时校正父亲遗传背景。

3.4 批量性状分析（--batch-trait）

功能说明

对多个性状同时进行方差组分估计和 GWAS 扫描。使用 DFREML（Derivative-Free REML）算法， 通过网格搜索最大化对数似然来估计每个性状的遗传方差（Vg）和残差方差（Ve）。

批量模式的重要限制：
  • 仅能包含1 个 GRM，不支持多个 GRM
  • 不支持额外随机效应（--rand-col / --rand-cov 不可用）
  • 不支持分块模式（--chunk 会被自动忽略）
  • 模型结构为：y = Xb + Zg + e（固定效应 + 加性遗传 + 残差）

输出文件

文件内容
.var每个性状的 Vg、Ve、标准误及遗传力（h²）
.rand每个性状的随机效应预测值（育种值）
.batchmlm所有性状的 Effect、SE、Pvalue（Vg、Ve—— 仅当 --test 为 exact 时生成）
.batchSummary每个性状的 Lambda GC 和 Mean Chi² 汇总
.log运行日志

专属参数

参数说明默认值
--grm PREFIX单个 GRM 文件前缀（必须恰好一个）—（必填）
--phe-col STR批量分析的性状列（逗号分隔）—
--test METHODGWAS 检验方法：standard、gamma、exact（注意：不支持 quasi-standard）standard
--gwas执行 GWAS 扫描关闭
--seed N随机数种子6666
--num-ranvec Ngamma 方法使用的随机向量数100
--re-gamma-size N估计 gamma 因子的随机 SNP 采样数500

完整示例

# 批量分析 3 个性状（第 2、3、4 列），仅估计方差组分
RBP --batch-trait \
  --grm my_grm \
  --phe phe.txt \
  --phe-col 2,3,4 \
  --fix-col 5 \
  --cov-col 6,7 \
  --geno ./data/geno \
  --out batch_gs \
  --ncpus 8

# 批量分析 + GWAS 扫描（standard 方法）
RBP --batch-trait \
  --grm my_grm \
  --phe phe.txt \
  --phe-col 2,3,4 \
  --fix-col 5 \
  --cov-col 6,7 \
  --geno ./data/geno \
  --gwas --test standard \
  --out batch_gwas \
  --ncpus 16

# 批量分析 + GWAS（gamma 近似方法，适合大样本快速扫描）
RBP --batch-trait \
  --grm my_grm \
  --phe phe.txt \
  --phe-col 2,3,4,5,6,7,8,9,10 \
  --geno ./data/geno \
  --gwas --test gamma \
  --num-ranvec 200 --re-gamma-size 1000 \
  --out batch_gamma_gwas \
  --ncpus 24
