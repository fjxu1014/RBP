User Manual of RBP

An efficient GWAS and GS analysis tool for large-scale genomic data based on the Mixed Linear Model (MLM), supporting batch-trait analysis.

1. Release

RBP is currently compiled and supported on the Linux x86_64 platform. To maximize calculation efficiency across different CPU architectures, we provide two pre-compiled executable versions linked with different linear algebra libraries:

  • MKL version: Highly recommended for Intel CPUs.
  • OpenBLAS version: Recommended for AMD CPUs or systems where MKL is not optimal.

2. Introduction

RBP (Rapid Batch-process-supported Powerful tool) is a high-performance computing tool designed for large-scale genomic data. Based on the Mixed Linear Model (MLM) framework, it is used for:

  • GWAS (Genome-Wide Association Study) — Scanning the genome for SNP loci significantly associated with traits.
  • GS (Genomic Selection) — Estimating breeding values (GEBV / Random Effects) for individual selection.

Core features of RBP:

  • High performance: Implemented in C++ + Armadillo + OpenMP, supporting multi-thread parallelism.
  • Double precision support: Supports both float (single-precision) and double (double-precision) calculations. Single precision saves significant memory.
  • Out-of-Core block mode: Supports block mapping for ultra-large datasets (individuals > 50,000), breaking memory limits.
  • Batch-trait analysis: Supports simultaneous variance component estimation and GWAS scanning for multiple traits in a single run.
  • Multiple random effects: Supports building multiple GRMs as random effects, as well as additional categorical random effects.
  • Multiple GWAS testing methods: Standard MLM, Quasi-Standard, Gamma approximation, Exact MLM.

RBP provides four sub-commands (modules):

  1. --make-grm — Build Genomic Relationship Matrix (GRM)
  2. --trans-grm — Convert GRM precision (Float <-> Double)
  3. --kpca — GRM-based Principal Component Analysis (kPCA)
  4. --single-trait — Single-trait GWAS / AI-REML analysis
  5. --batch-trait — Batch multi-trait GWAS / DFREML analysis

3. Input File Formats

3.1. Phenotype file

The phenotype file is a plain text file, with columns separated by spaces, tabs, or commas.

Must satisfy: The file must contain a header, and the first column must be the individual ID.

File example (phe.txt):

ID          Trait1   Trait2   Sex    Batch   Age    Sire
IND001      12.5     8.3      M      B1      2.1    BULL_A
IND002      14.1     9.1      F      B2      1.8    BULL_B
IND003      11.8     7.9      M      B1      2.4    BULL_A
IND004      13.2     8.7      F      B1      1.5    BULL_C
IND005      10.9     7.2      M      B2      2.0    BULL_B
...

Common parameters related to the phenotype file:

ParameterDescriptionExample
--phe FILE Specify the phenotype file path --phe phe.txt
--phe-col STR Specify the column(s) as the response variable (trait), 1-based index, comma-separated. Single-trait mode only takes the first column; batch mode can take multiple columns. --phe-col 2 or --phe-col 2,3
--fix-col STR Specify categorical fixed effect columns (e.g., sex, batch), 1-based, comma-separated. The program automatically performs dummy variable encoding (using the last level as a reference). --fix-col 4,5
--cov-col STR Specify continuous covariate columns (e.g., age, weight), 1-based, comma-separated. --cov-col 6
--rand-col STR Specify the columns of categorical variables for additional random effects, 1-based, comma-separated (only supported in single-trait mode). --rand-col 7
--rand-cov LIST Specify the covariance matrix file prefix corresponding to additional random effects (space-separated). '1' indicates an identity matrix (i.i.d. random effect). A .txt list file is also accepted. --rand-cov pedigree_grm or --rand-cov 1
NOTE

Column index note: All column indices (--phe-col, --fix-col, --cov-col, --rand-col) are 1-based, meaning the first column (ID column) is 1.

3.2. Genotype file

The genotype file uses the PLINK binary format (.bed / .bim / .fam). Specify the prefix (without extension, or with the .bed suffix) using the --geno parameter.

Parameters for specifying individual and SNP subsets:

ParameterDescriptionExample
--geno FILE Genotype data PLINK prefix (or full .bed path) --geno ./data/geno
--geno-ind STR Specify the individual indices to read (1-based), supporting a comma-separated list OR a file path with one number per line. --geno-ind 1,3,5,100 or --geno-ind keep_ids.txt
--geno-snp STR Specify the SNP indices to read (1-based), supporting a comma-separated list OR a file path with one number per line. --geno-snp 1,2,3,500 or --geno-snp snp_list.txt
Smart parsing: --geno-ind and --geno-snp will first try to open the input as a file path; if the file does not exist, it will be parsed as a comma-separated list. Indices are 1-based.

Index file example (e.g., keep_ids.txt or snp_list.txt):

The file is a single-column plain text format, with one number per line, representing the index (1-based) of the individual or SNP to keep from the original data.

1
3
5
102
105
...

4. Modules

RBP's modules are selected by the first positional parameter. The following common parameters apply to all modules:

ParameterDescriptionDefault
--out PREFIXOutput file prefixrbp
--ncpus NNumber of CPU threads1
--floatUse single-precision (float) calculation to save memory. If enabled during GRM construction, subsequent analysis must also enable itOff (uses double)
--chunkEnable Out-of-Core block mode (recommended for individuals > 50,000)Off
--quietQuiet mode, do not print logs to the terminal (logs are still written to the .log file)Off
Important: Consistency of the --float parameter
If --float is used during GRM construction, subsequent modules like kPCA and single-trait analysis MUST also use --float. Mixing float and double GRM binary files will cause reading errors due to inconsistent data types.

4.1. Make GRM (--make-grm)

Description

Build the Genomic Relationship Matrix (GRM) from PLINK binary genotype data. Supports three calculation methods and the option to calculate the generalized inverse matrix of the GRM.

Output Files

ExtensionContent
.GA.binAdditive GRM binary matrix (Yang or VanRaden method)
.GD.binDominance GRM binary matrix
.idIndividual ID list corresponding to the rows/columns of the GRM
.inv.txtGeneralized inverse matrix of the GRM, three-column format (row, col, val) — generated only when --inv is specified

Exclusive Parameters

ParameterDescriptionDefault
--method NGRM construction method: 1=Yang additive (default), 2=VanRaden additive, 3=Dominance1
--weight FILESNP weight file (single-column text, each line corresponds to a SNP weight)None (all SNPs equal weight)
--snp-chunk NSNP chunk size processed at a time, affecting memory usage and calculation efficiency10000
--ind-chunk NIndividual chunk size processed at a time in block mode (effective only under --chunk mode)512
--invCalculate the generalized inverse matrix of the GRM and save it in three-column format .inv.txtOff
--geno-ind STRSelect individual subset for GRM constructionAll individuals
--geno-snp STRSelect SNP subset for GRM constructionAll SNPs

SNP weight file example (snp_weights.txt):

Single-column text file, the number of lines must exactly match the number of SNPs finally used in the calculation, each line corresponds to a weight value of a SNP (negative values are not supported).

1.0
0.8
1.5
2.0
1.1
...

Complete Examples

# Build additive GRM using the default Yang method (double precision)
RBP --make-grm --geno ./data/geno --out my_grm --ncpus 8

# Use VanRaden method + SNP weights (single precision, suitable for ultra-large datasets)
RBP --make-grm --geno ./data/geno --out my_grm_vanraden --method 2 --weight snp_weights.txt --float --ncpus 16

# Build dominance effect GRM and calculate the inverse matrix
RBP --make-grm --geno ./data/geno --out dom_grm --method 3 --inv --ncpus 8

# Build GRM using a specified individual subset, enable block mode (individuals > 50,000)
RBP --make-grm --geno ./data/geno --out large_grm --geno-ind keep_list.txt --chunk --ind-chunk 1024 --ncpus 24

4.2. GRM Precision Conversion (--trans-grm)

Description

Automatically detect the data precision of a GRM binary file (double / 64-bit or float / 32-bit) and perform a high-speed conversion to the other format.

This module utilizes a highly optimized OpenMP multi-threaded streaming architecture with O(1) memory complexity. Regardless of whether the matrix is 1GB or 500GB, the physical memory consumption is strictly locked at around 12 MB, completely avoiding Out-of-Memory (OOM) errors. The accompanying plain text files (.id and .col) are safely duplicated to the new prefix.

Safety Protection: The --out prefix CANNOT be identical to the input --grm prefix. This is a strict safeguard to prevent irreversible data destruction via in-place file truncation.

Output Files

ExtensionContent
.binThe converted GRM binary matrix (Double -> Float, or Float -> Double)
.idCopied individual ID list
.colCopied column list (Generated only if the input is a compressed GRM)

Exclusive Parameters

ParameterDescriptionDefault
--grm PREFIXInput GRM file prefix (without the .bin extension)— (Required)
Smart Detection: You do not need to specify whether the input is float or double. RBP will calculate the exact dimensions from the .id (and .col) files and use underlying OS calls to determine the byte size, automatically executing dimension reduction (truncation) or precision promotion (zero-padding).

Complete Example

# Automatically convert a GRM (e.g., from double to float to save memory for subsequent steps)
RBP --trans-grm \
  --grm my_huge_grm \
  --out my_huge_grm_float \
  --ncpus 16
Recommendation: It is highly recommended to use double-precision GRM (or use --trans-grm to convert a float GRM to double) in subsequent downstream analyses to ensure optimal precision and numerical stability. This is especially crucial when using the Conjugate Gradient (CG) solver (--solver CG) in the --single-trait module, as the CG algorithm is sensitive to floating-point truncation errors.

4.3. kPCA Analysis (--kpca)

Description

Based on the constructed GRM, use the Lanczos algorithm to calculate the top k largest eigenvalues and their corresponding eigenvectors (i.e., principal components, PCs). kPCA is commonly used for population stratification detection and introducing PCs as covariates into GWAS models to control population structure.

Output Files

ExtensionContent
.valEigenvalue list (one per line)
.vecPrincipal component (PCs) matrix: the first column is the individual ID, and subsequent columns are PC scores
Note: If multiple GRMs are provided (--grm file1 file2), the kPCA module will only analyze the first GRM and output a warning.

Exclusive Parameters

ParameterDescriptionDefault
--grm PREFIXInput GRM file prefix (without the .bin extension)— (Required)
--npc NNumber of principal components to extract10
--tol VALLanczos algorithm convergence tolerance1e-6

Complete Examples

# Extract the top 10 principal components from the GRM
RBP --kpca --grm my_grm --npc 10 --ncpus 8

# Specify output prefix, extract top 20 PCs, and use single precision
RBP --kpca --grm my_grm_float --npc 20 --out pca_result --float --ncpus 8

4.4. Single-trait Analysis (--single-trait)

Description

Fit a mixed linear model for a single trait, use the AI-REML (Average Information Restricted Maximum Likelihood) algorithm to estimate variance components, and optionally perform GWAS scanning or calculate random effect predicted values (breeding values) and their PEV (Prediction Error Variance).

Single-trait analysis supports various complex model structures:

  • General GS / GWAS (Single GRM + Residuals)
  • Repeatability model (repeated measures, use --repeat)
  • Multiple GRM random effects (e.g., additive + dominance + epistasis)
  • Including additional categorical random effects (e.g., batch, maternal effects)
  • Using parental ancestral genetic random effects as background (use --rand-col + --rand-cov)

Output Files

FileContent
.varVariance component estimates (Variance, SE, Proportion)
.betaFixed effect estimates and standard errors
.randPredicted values of all random effects (breeding values, etc.), optionally PEV and reliability r²
.mlmGWAS scan results (SNP, Chr, Pos, Effect, SE, Pvalue) — generated only when --gwas is specified
.logRun log (including Lambda GC and Mean Chi² statistics)

Exclusive Parameters

ParameterDescriptionDefault
--grm LISTOne or more GRM file prefixes (space-separated). A .txt list file is also accepted. The data types (float/double) of all GRMs must be consistent—
--rand-col STRCategorical variable columns for additional random effects—
--rand-cov LISTCovariance matrix file prefixes for additional random effects (space-separated). If '1', it means i.i.d.—
--test METHODGWAS testing method: standard (Standard MLM), quasi-standard (not applicable to batch), gamma (Gamma approximation), exact (Exact test, not applicable to single)standard
--gwasPerform GWAS scanOff
--solver METHODAI-REML solver: CG (Conjugate Gradient, default) or CholeskyCG
--criterion NAI-REML convergence criteria:
0=Converged if either 1 or 2 is satisfied
1=Relative error of variance component estimates between two iterations < tol
2=Log-likelihood gradient of the objective function < tol		0
--momEnable MoM-RHE initialization of variance components (accelerates convergence)Off
--pevCalculate PEV and reliability (r²) of random effect predictionsOff
--repeatSpecify repeatability model (repeated measures)Off
--seed NRandom number seed6666
--num-ranvec NNumber of random vectors used by quasi-standard/gamma method100
--re-gamma-size NNumber of randomly sampled SNPs when estimating the gamma factor500
--re-est-gammaForce re-estimation of the gamma factorOff
--maxit NAI-REML maximum iterations30
--tol VALAI-REML convergence tolerance1e-5
--cg-maxit NCG solver maximum iterations500
--cg-tol VALCG solver convergence tolerance1e-6

Detailed usage of the --grm parameter

Multiple GRM data types must be consistent: When specifying multiple GRMs, all GRMs must be constructed with the same precision (all float or all double), otherwise data reading errors will occur.

Three ways to specify multiple GRMs:

# Method 1: List multiple GRM prefixes directly (space-separated)
RBP --single-trait --grm grm_add grm_dom grm_epi --phe phe.txt --phe-col 2 --geno geno

# Method 2: Specify via a .txt list file (one GRM prefix per line)
# Content of grm_list.txt file:
#   grm_add
#   grm_dom
#   grm_epi
RBP --single-trait --grm grm_list.txt --phe phe.txt --phe-col 2 --geno geno

# Method 3: Mixed use (direct prefixes + list file)
RBP --single-trait --grm grm_add grm_list.txt --phe phe.txt --phe-col 2 --geno geno

GRM data type requirements:

  • All GRMs must be constructed with the same precision (all float or all double)
  • If --float was used during GRM construction, --float must also be used in single-trait analysis
  • Each GRM prefix must include .bin and .id files; if it is a compressed matrix, a .col file is also required

Complete Examples

Scenario 1: General GS / GWAS (Single GRM)
# Only estimate variance components and breeding values (GS)
RBP --single-trait \
  --grm my_grm \
  --phe phe.txt \
  --phe-col 2 \
  --fix-col 4 \
  --cov-col 6 \
  --geno ./data/geno \
  --out gs_result \
  --ncpus 8

# GWAS scan (Standard MLM)
RBP --single-trait \
  --grm my_grm \
  --phe phe.txt \
  --phe-col 2 \
  --fix-col 4 \
  --cov-col 6 \
  --geno ./data/geno \
  --gwas \
  --out gwas_result \
  --ncpus 8

# GS + GWAS and calculate prediction error variance of estimated breeding values
RBP --single-trait \
  --grm my_grm \
  --phe phe.txt \
  --phe-col 2 \
  --fix-col 4 \
  --cov-col 6 \
  --geno ./data/geno \
  --gwas --pev \
  --out gs_gwas_result \
  --ncpus 8
Scenario 2: GS / GWAS for repeatability traits
# When there are repeated measure records for the same trait, use --repeat to specify the repeatability model
RBP --single-trait \
  --grm my_grm \
  --phe phe_repeat.txt \
  --phe-col 2 \
  --fix-col 4,5 \
  --geno ./data/geno \
  --gwas --repeat \
  --out repeat_result \
  --ncpus 8
Scenario 3: GS / GWAS including multiple GRM random effects
# Incorporate both additive and dominance GRMs into the model
RBP --single-trait \
  --grm grm_add grm_dom \
  --phe phe.txt \
  --phe-col 2 \
  --fix-col 4 \
  --cov-col 6 \
  --geno ./data/geno \
  --gwas --pev \
  --out multi_grm_result \
  --ncpus 8
Scenario 4: GS / GWAS using parental ancestral genetic random effects as background
# --rand-col specifies the column index in the phenotype file for the individual-to-parent mapping
# --rand-cov specifies the parental relationship matrix
RBP --single-trait \
  --grm my_grm \
  --phe phe_with_sire.txt \
  --phe-col 3 \
  --fix-col 5 \
  --rand-col 7 \
  --rand-cov sire_grm \
  --geno ./data/geno \
  --gwas --pev \
  --out sire_model_result \
  --ncpus 8
Parental model example note: Column 7 (--rand-col 7) in the phenotype file records the sire ID corresponding to each individual; sire_grm is the pre-constructed relationship matrix (GRM) among sires. The model incorporates the sire genetic effect as an additional random effect, thereby correcting the sire genetic background when estimating individual additive effects.

4.5. Batch-trait Analysis (--batch-trait)

Description

Simultaneously perform variance component estimation and GWAS scanning for multiple traits. Uses the DFREML (Derivative-Free REML) algorithm to estimate the genetic variance (Vg) and residual variance (Ve) of each trait by maximizing the log-likelihood via grid search.

Important limitations of batch mode:
  • Can only include 1 GRM, multiple GRMs are not supported
  • Additional random effects are not supported (--rand-col / --rand-cov are disabled)
  • Block mode is not supported (--chunk will be automatically ignored)
  • Model structure: y = Xb + Zg + e (Fixed effects + Additive genetic + Residuals)

Output Files

FileContent
.varVg, Ve, SE, and heritability (h²) for each trait
.randRandom effect predicted values (breeding values) for each trait
.batchmlmEffect, SE, Pvalue of all traits (Vg, Ve — generated only when --test is exact)
.batchSummaryLambda GC and Mean Chi² summary for each trait
.logRun log

Exclusive Parameters

ParameterDescriptionDefault
--grm PREFIXSingle GRM file prefix (must be exactly one)— (Required)
--phe-col STRTrait columns for batch analysis (comma-separated)—
--test METHODGWAS testing method: standard, gamma, exact (Note: quasi-standard is not supported)standard
--gwasPerform GWAS scanOff
--seed NRandom number seed6666
--num-ranvec NNumber of random vectors used by the gamma method100
--re-gamma-size NNumber of random SNPs sampled to estimate the gamma factor500

Complete Examples

# Batch analysis for 3 traits (columns 2, 3, 4), only estimate variance components
RBP --batch-trait \
  --grm my_grm \
  --phe phe.txt \
  --phe-col 2,3,4 \
  --fix-col 5 \
  --cov-col 6,7 \
  --geno ./data/geno \
  --out batch_gs \
  --ncpus 8

# Batch analysis + GWAS scan (standard method)
RBP --batch-trait \
  --grm my_grm \
  --phe phe.txt \
  --phe-col 2,3,4 \
  --fix-col 5 \
  --cov-col 6,7 \
  --geno ./data/geno \
  --gwas --test standard \
  --out batch_gwas \
  --ncpus 16

# Batch analysis + GWAS (gamma approximation method, suitable for fast scanning of large samples)
RBP --batch-trait \
  --grm my_grm \
  --phe phe.txt \
  --phe-col 2,3,4,5,6,7,8,9,10 \
  --geno ./data/geno \
  --gwas --test gamma \
  --num-ranvec 200 --re-gamma-size 1000 \
  --out batch_gamma_gwas \
  --ncpus 24
